Lua String Pattern Reference

Lua patterns are similar to regex but use % as the escape character instead of \. All standard character classes use % prefix.

Lua Syntax Quick Guide

Common Lua syntax rules that the validator checks — and frequent mistakes that cause runtime errors.

Block keywords (must match)

  • if/then ... end — every if needs a matching end
  • function ... end — including anonymous functions
  • do ... end — explicit scope blocks
  • while ... do ... end — loop condition + body
  • for ... do ... end — numeric and generic for
  • repeat ... until condition — different end keyword!

Common Lua mistakes

  • String concatenation: use .. not + — "a" .. "b"
  • Not-equal: ~= not !=
  • Nil check: if x ~= nil or just if x then
  • Table length: #t only reliable for sequences
  • Global vs local: always use local — globals pollute _G
  • No continue: use goto continue (Lua 5.2+) or restructure

Roblox Luau differences

  • Types: Luau adds optional typing — local x: number = 5
  • String interpolation: \`Hello {name}\` (Luau-only)
  • Compound assignment: +=, -=, *= (Luau-only)
  • continue: available in Luau loops (not standard Lua 5.1)
  • bit32 library: built-in, no require needed

OpenResty / Nginx Lua

  • ngx.say() vs print(): use ngx.say() for HTTP response output
  • Coroutine-based: every request is a coroutine — avoid blocking calls
  • No require for built-ins: ngx, cjson available globally
  • Phase restrictions: not all ngx APIs work in all request phases
  • Shared dict: use ngx.shared.DICT for cross-request state

Lua Tools – FAQ

How do I validate Lua syntax online?+
Paste your Lua code in the Validator tab and click Validate. The tool checks for: unmatched block keywords (every if needs an end, repeat needs until), unbalanced parentheses and square brackets, unclosed string literals, and common style issues. Each error shows the exact line number. The validator catches structural errors — for full semantic analysis (type checking, undefined variables), use tools like Luacheck or Stella.
What is the difference between Lua patterns and regular expressions?+
Lua patterns use % as the escape character instead of \. Key differences: %d for digits (not \d), %a for letters (not [a-zA-Z]), %s for whitespace (not \s). Lua patterns have no | (alternation) operator — you have to handle alternatives with multiple matches or captures. The lazy quantifier is - (not *?). There are no lookaheads or lookbehinds. Lua patterns are simpler and faster than full regex but less powerful. Use the Pattern Tester tab above to experiment interactively.
How do I minify Lua scripts for production?+
The Minifier tab removes comments (single-line -- and block --[[ ]]) and reduces whitespace to the minimum needed for valid syntax. This reduces file size typically 20-50% for well-commented code. For Lua in game engines (Roblox, Love2D): minification reduces script transfer size and can slightly improve initial parse time. For OpenResty/Nginx: LuaJIT compiles scripts on first run, so minification has minimal runtime impact but reduces startup memory. Never minify code you're still actively debugging — always keep the original source.
How do I test Lua string patterns?+
The Pattern Tester tab simulates string.find, string.match, string.gmatch, and string.gsub. Enter your pattern and test string to see which parts match. Captures (using parentheses in the pattern) are shown separately. The JavaScript simulation closely matches Lua pattern behavior but note: Lua patterns are not regex. Some edge cases in %b (balanced match) and frontier patterns %f[ ] are not fully simulated. For precise testing in a live Lua environment, use the Lua REPL at replit.com or a local Lua installation.
